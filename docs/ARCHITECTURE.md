# Inkwell Architecture

## Overview

Inkwell is a hybrid local/cloud AI word processor built on:

- **TipTap (ProseMirror)** вҖ” Rich text editor core
- **Next.js** вҖ” Web application framework
- **Tauri** вҖ” Desktop shell with native capabilities
- **Claude API** вҖ” Cloud AI inference
- **llama.cpp / whisper.cpp** вҖ” Local AI inference

## Key Packages

### @inkwell/editor
TipTap editor core with custom extensions:
- **Ghost Text** вҖ” AI inline suggestions as decorations
- **Slash Commands** вҖ” "/" trigger for AI operations
- **Diff Preview** вҖ” Before/after rendering for rewrites
- **AI Undo** вҖ” Atomic undo for multi-step AI edits
- **Collaboration** вҖ” Y.js real-time editing

### @inkwell/document-ai
The AI brain:
- **Router** вҖ” Routes operations to local or cloud models
- **Queue** вҖ” Priority queue with cancellation and budgets
- **Context** вҖ” Assembles prompt context with caching
- **Reconciler** вҖ” Converts AI output to editor transactions
- **Claude Client** вҖ” Streaming API client

### @inkwell/mcp-workspace
MCP context server:
- Workspace file indexing and search
- Document analysis tools
- Style guide extraction

### @inkwell/shared
Shared types, constants, and utilities.

## Data Flow

1. User types in editor
2. Debounced trigger fires after 500ms idle
3. DocumentAI router selects model (local vs cloud)
4. Context manager assembles prompt with cached prefix
5. Request enters priority queue
6. Model generates response (streamed)
7. Reconciler maps output to ProseMirror transactions
8. Ghost text or diff preview renders result
9. User accepts or dismisses
